    1.1. ¬øQu√© es el Aprendizaje No Supervisado?

    El Aprendizaje No Supervisado es una rama del Machine Learning donde los algoritmos trabajan con datos sin etiquetas. A diferencia del aprendizaje supervisado, no existe una "respuesta correcta" predefinida que gu√≠e el entrenamiento.

    Definiciones Clave

    • Objetivo principal: Descubrir estructuras ocultas, patrones o agrupaciones inherentes en los datos que no son evidentes a simple vista.

    • Analog√≠a: Imagina que te dan una caja con miles de fotograf√≠as sin ninguna descripci√≥n. El aprendizaje no supervisado ser√≠a como organizarlas autom√°ticamente en grupos (paisajes, retratos, animales, etc.) bas√°ndose √∫nicamente en las similitudes visuales entre ellas.

    Diferencias con el Aprendizaje Supervisado

    Aspecto Supervisado No Supervisado
    Datos Etiquetados (X, y) Sin etiquetas (solo X)
    Objetivo Predecir una variable objetivo Descubrir estructura en los datos
    Evaluaci√≥n M√©tricas claras (accuracy, F1, MSE) M√©tricas indirectas (silueta, inercia)
    Ejemplos Clasificaci√≥n, Regresi√≥n Clustering, Reducci√≥n de dimensionalidad
    Feedback Conocemos si la predicci√≥n es correcta No hay "respuesta correcta"

    Tipos de Problemas No Supervisados

    El aprendizaje no supervisado abarca principalmente cuatro tipos de problemas:

    1. Clustering (Agrupamiento):
    2. Objetivo: Dividir los datos en grupos (clusters) donde los elementos dentro de un grupo son similares entre s√≠ y diferentes a los de otros grupos.
    3. Algoritmos: K-Means, DBSCAN, Clustering Jer√°rquico, OPTICS, Mean Shift.

    4. Aplicaci√≥n: Segmentaci√≥n de clientes, agrupaci√≥n de documentos.

    5. Reducci√≥n de Dimensionalidad:

    6. Objetivo: Reducir el n√∫mero de variables (features) manteniendo la mayor cantidad de informaci√≥n posible.
    7. Algoritmos: PCA, t-SNE, UMAP, LDA, Autoencoders.

    8. Aplicaci√≥n: Visualizaci√≥n de datos, compresi√≥n, preprocesamiento.

    9. Detecci√≥n de Anomal√≠as:

    10. Objetivo: Identificar puntos de datos que se desv√≠an significativamente del comportamiento normal.
    11. Algoritmos: Isolation Forest, One-Class SVM, LOF (Local Outlier Factor).

    12. Aplicaci√≥n: Detecci√≥n de fraudes, mantenimiento predictivo.

    13. Reglas de Asociaci√≥n:

    14. Objetivo: Descubrir relaciones interesantes entre variables en grandes conjuntos de datos.
    15. Algoritmos: Apriori, FP-Growth, Eclat.
    16. Aplicaci√≥n: An√°lisis de cesta de compra, sistemas de recomendaci√≥n.

    1.4. Preprocesamiento de Datos para Aprendizaje No Supervisado

    El preprocesamiento es a√∫n m√°s cr√≠tico en el aprendizaje no supervisado que en el supervisado, ya que los algoritmos son muy sensibles a la escala y calidad de los datos.

    1.4.1. Escalado de Caracter√≠sticas

    ¬øPor qu√© es obligatorio?

    La mayor√≠a de algoritmos no supervisados se basan en medidas de distancia (Euclidiana, Manhattan, etc.). Si las variables tienen escalas muy diferentes, las de mayor magnitud dominar√°n completamente el c√°lculo.

    Ejemplo del problema: 

    Cliente A: Edad=25, Salario=50000
Cliente B: Edad=35, Salario=51000
Cliente C: Edad=26, Salario=80000
    Sin escalado, la diferencia de salario (miles) dominar√° sobre la edad (decenas).

    M√©todos de Escalado:

    from sklearn.preprocessing import StandardScaler, MinMaxScaler, RobustScaler

# 1. Estandarizaci√≥n (Z-score) - El m√°s com√∫n

    ¬øCu√°ndo usar cada uno?

    1.5. M√©tricas de Evaluaci√≥n en Aprendizaje No Supervisado

    Evaluar modelos no supervisados es m√°s complejo porque no hay etiquetas de referencia. Existen dos tipos de m√©tricas:

    1.5.1. M√©tricas Internas (sin etiquetas reales)

    Eval√∫an la calidad del clustering bas√°ndose √∫nicamente en los datos y las asignaciones de cluster.

    Coeficiente de Silueta (Silhouette Score)

    Mide qu√© tan similar es un punto a su propio cluster comparado con otros clusters.

    F√≥rmula para un punto \(i\): \(\(s(i) = \frac{b(i) - a(i)}{\max(a(i), b(i))}\)\)

    Donde: - \(a(i)\) = distancia media de \(i\) a los otros puntos de su mismo cluster (cohesi√≥n) - \(b(i)\) = distancia media m√≠nima de \(i\) a los puntos del cluster m√°s cercano (separaci√≥n)

    Interpretaci√≥n: - \(s(i) \approx 1\): El punto est√° bien asignado a su cluster - \(s(i) \approx 0\): El punto est√° en la frontera entre clusters - \(s(i) < 0\): El punto probablemente est√° mal asignado

    from sklearn.metrics import silhouette_score, silhouette_samples

# Silueta promedio del clustering
score = silhouette_score(X, labels)
print(f"Silhouette Score: {score:.3f}")

# Silueta por cada muestra (para an√°lisis detallado)
sample_scores = silhouette_samples(X, labels)

    Inercia (Within-Cluster Sum of Squares - WCSS)

    Suma de las distancias al cuadrado de cada punto al centroide de su cluster. Solo para K-Means.

    \[WCSS = \sum_{i=1}^{k}\sum_{x \in C_i} ||x - \mu_i||^2\]

    Interpretaci√≥n: - Menor inercia = clusters m√°s compactos - Se usa en el M√©todo del Codo para encontrar el n√∫mero √≥ptimo de clusters

    # La inercia se obtiene directamente del modelo KMeans

kmeans.fit(X)
print(f"Inercia: {kmeans.inertia_}")

    √çndice Calinski-Harabasz (Variance Ratio Criterion)

    Ratio entre la dispersi√≥n entre clusters y la dispersi√≥n dentro de clusters.

    \[CH = \frac{SS_B / (k-1)}{SS_W / (n-k)}\]

    Donde: - \(SS_B\) = dispersi√≥n entre clusters - \(SS_W\) = dispersi√≥n dentro de clusters - \(k\) = n√∫mero de clusters - \(n\) = n√∫mero de muestras

    Interpretaci√≥n: Mayor valor = mejor clustering

    from sklearn.metrics import calinski_harabasz_score

score = calinski_harabasz_score(X, labels)
print(f"Calinski-Harabasz Score: {score:.3f}")

    √çndice Davies-Bouldin

    Mide la similitud promedio entre cada cluster y su cluster m√°s similar.

    \[DB = \frac{1}{k}\sum_{i=1}^{k}\max_{j \neq i}\left(\frac{s_i + s_j}{d_{ij}}\right)\]

    Donde: - \(s_i\) = dispersi√≥n media del cluster \(i\) - \(d_{ij}\) = distancia entre centroides de clusters \(i\) y \(j\)

    Interpretaci√≥n: Menor valor = mejor clustering (clusters m√°s separados y compactos)

    from sklearn.metrics import davies_bouldin_score

score = davies_bouldin_score(X, labels)
print(f"Davies-Bouldin Score: {score:.3f}")

    1.5.2. M√©tricas Externas (con etiquetas reales)

    Cuando disponemos de etiquetas reales (ground truth), podemos comparar los clusters descubiertos con las clases verdaderas.

    Adjusted Rand Index (ARI)

    Mide la similitud entre dos asignaciones de clusters, ajustada por azar.

    from sklearn.metrics import adjusted_rand_score

# Rango: [-1, 1], donde 1 = asignaci√≥n perfecta

    Normalized Mutual Information (NMI)

    Mide la informaci√≥n mutua entre las asignaciones, normalizada.

    from sklearn.metrics import normalized_mutual_info_score

# Rango: [0, 1], donde 1 = asignaci√≥n perfecta

    M√©trica Tipo Rango Mejor valor Uso principal

    Inercia (WCSS) Interna [0, ‚àû) Menor M√©todo del codo

    1.6. El M√©todo del Codo (Elbow Method)

    Es la t√©cnica m√°s popular para determinar el n√∫mero √≥ptimo de clusters en K-Means.

    Concepto

    1. Ejecutar K-Means con diferentes valores de \(k\) (n√∫mero de clusters)
    2. Para cada \(k\), calcular la inercia (WCSS)
    3. Graficar \(k\) vs. inercia
    4. Buscar el "codo": el punto donde la reducci√≥n de inercia se desacelera significativamente

    1.7.2. Gr√°fico de Silueta

    from sklearn.metrics import silhouette_samples
import numpy as np

def plot_silhouette(X, labels, n_clusters):
    """Grafica el an√°lisis de silueta por cluster."""
    fig, ax = plt.subplots(figsize=(10, 8))

    silhouette_avg = silhouette_score(X, labels)
    sample_silhouette_values = silhouette_samples(X, labels)

    y_lower = 10
    for i in range(n_clusters):
        # Valores de silueta para el cluster i
        cluster_silhouette_values = sample_silhouette_values[labels == i]
        cluster_silhouette_values.sort()

        cluster_size = cluster_silhouette_values.shape[0]
        y_upper = y_lower + cluster_size

        color = plt.cm.viridis(float(i) / n_clusters)
        ax.fill_betweenx(np.arange(y_lower, y_upper),
                         0, cluster_silhouette_values,
                         facecolor=color, edgecolor=color, alpha=0.7)

        ax.text(-0.05, y_lower + 0.5 * cluster_size, str(i))
        y_lower = y_upper + 10

    ax.axvline(x=silhouette_avg, color="red", linestyle="--", 
               label=f'Silueta media: {silhouette_avg:.3f}')
    ax.set_xlabel("Coeficiente de Silueta")
    ax.set_ylabel("Cluster")
    ax.legend()
    plt.title("An√°lisis de Silueta")
    plt.show()

    1.10. Buenas Pr√°cticas

    ‚úÖ Hacer siempre:

    1. Escalar los datos antes de aplicar algoritmos basados en distancias
    2. Explorar los datos visualmente antes del clustering (EDA)
    3. Probar m√∫ltiples algoritmos y comparar resultados
    4. Usar m√∫ltiples m√©tricas para evaluar la calidad
    5. Validar los resultados con conocimiento del dominio
    6. Documentar las decisiones (por qu√© se eligi√≥ cierto k, algoritmo, etc.)

    ‚ùå Evitar:

    1. Asumir que existe una estructura de clusters cuando puede no haberla
    2. Confiar ciegamente en una sola m√©trica
    3. Ignorar outliers sin investigarlos
    4. Aplicar algoritmos sin entender sus supuestos
    5. Sobrevalorar el n√∫mero de clusters (m√°s no siempre es mejor)

    1.11. Ejercicio Integrador: Pipeline Completo

    """
Pipeline completo de clustering con evaluaci√≥n
"""
import numpy as np
import pandas as pd
import matplotlib.pyplot as plt
from sklearn.preprocessing import StandardScaler
from sklearn.cluster import KMeans
from sklearn.metrics import silhouette_score, calinski_harabasz_score, davies_bouldin_score
from sklearn.datasets import load_iris

# 1. CARGAR DATOS
iris = load_iris()
X = iris.data
feature_names = iris.feature_names

print("="*50)
print("PIPELINE DE CLUSTERING NO SUPERVISADO")
print("="*50)

# 2. PREPROCESAMIENTO
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)
print(f"\n[1] Datos escalados: {X_scaled.shape}")

# 3. M√âTODO DEL CODO
inertias = []
silhouettes = []
K_range = range(2, 11)

for k in K_range:
    kmeans = KMeans(n_clusters=k, random_state=42, n_init=10)
    kmeans.fit(X_scaled)
    inertias.append(kmeans.inertia_)
    silhouettes.append(silhouette_score(X_scaled, kmeans.labels_))

# Visualizar
fig, axes = plt.subplots(1, 2, figsize=(14, 5))

axes[0].plot(K_range, inertias, 'bo-', linewidth=2)
axes[0].set_xlabel('N√∫mero de Clusters (k)')
axes[0].set_ylabel('Inercia')
axes[0].set_title('M√©todo del Codo')
axes[0].grid(True, alpha=0.3)

axes[1].plot(K_range, silhouettes, 'go-', linewidth=2)
axes[1].set_xlabel('N√∫mero de Clusters (k)')
axes[1].set_ylabel('Silhouette Score')
axes[1].set_title('Silueta vs k')
axes[1].grid(True, alpha=0.3)

plt.tight_layout()
plt.show()

# 4. MODELO FINAL (k=3 basado en an√°lisis)
k_optimo = 3
kmeans_final = KMeans(n_clusters=k_optimo, random_state=42, n_init=10)
labels = kmeans_final.fit_predict(X_scaled)

print(f"\n[2] Clustering con k={k_optimo}")
print(f"    Distribuci√≥n de clusters: {np.bincount(labels)}")

# 5. EVALUACI√ìN
print(f"\n[3] M√âTRICAS DE EVALUACI√ìN:")
print(f"    - Silhouette Score:      {silhouette_score(X_scaled, labels):.4f}")
print(f"    - Calinski-Harabasz:     {calinski_harabasz_score(X_scaled, labels):.4f}")
print(f"    - Davies-Bouldin:        {davies_bouldin_score(X_scaled, labels):.4f}")

# 6. AN√ÅLISIS DE RESULTADOS
print(f"\n[4] AN√ÅLISIS POR CLUSTER:")
df = pd.DataFrame(X, columns=feature_names)
df['cluster'] = labels
print(df.groupby('cluster').mean().round(2))
