    Los Transformers han revolucionado el NLP desde su introducciÃ³n en 2017. Esta arquitectura es la base de modelos como BERT, GPT, T5 y los modernos Large Language Models (LLMs).

    7.1. IntroducciÃ³n a los Transformers

    El Problema con las RNN

    Antes de los Transformers, las Redes Neuronales Recurrentes (RNN) y sus variantes (LSTM, GRU) eran el estÃ¡ndar para procesamiento de secuencias. Sin embargo, tenÃ­an limitaciones:

    • Procesamiento secuencial: No pueden paralelizarse eficientemente.
    • Memoria a largo plazo: Dificultad para capturar dependencias lejanas.
    • Entrenamiento lento: Debido a la naturaleza secuencial.

    La SoluciÃ³n: AtenciÃ³n

    El paper "Attention Is All You Need" (Vaswani et al., 2017) introdujo el mecanismo de Self-Attention que permite:

    • ParalelizaciÃ³n completa: Procesa toda la secuencia a la vez.
    • Contexto global: Cada token puede atender a cualquier otro token.
    • Escalabilidad: Permite entrenar modelos muy grandes.

    Self-Attention

    El mecanismo de Self-Attention permite que cada token "atienda" a todos los demÃ¡s tokens de la secuencia.

    Para cada token, se calculan tres vectores:

    • Query (Q): Lo que este token estÃ¡ "buscando"
    • Key (K): Lo que este token "ofrece"
    • Value (V): La informaciÃ³n que contiene

    La fÃ³rmula de atenciÃ³n:

    \[\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V\]

    Donde \(d_k\) es la dimensiÃ³n de las keys.

    Multi-Head Attention

    En lugar de una sola atenciÃ³n, se usan mÃºltiples "cabezas" de atenciÃ³n que aprenden diferentes tipos de relaciones:

    \[\text{MultiHead}(Q, K, V) = \text{Concat}(\text{head}_1, ..., \text{head}_h)W^O\]

    Donde cada cabeza:

    \[\text{head}_i = \text{Attention}(QW_i^Q, KW_i^K, VW_i^V)\]

    Positional Encoding

    Como los Transformers no tienen nociÃ³n inherente de posiciÃ³n, se aÃ±ade informaciÃ³n posicional mediante:

    \[PE_{(pos, 2i)} = \sin(pos / 10000^{2i/d_{model}})$$ $$PE_{(pos, 2i+1)} = \cos(pos / 10000^{2i/d_{model}})\]

    7.3. Tipos de Modelos Transformer

    Encoder-only (BERT)

    • Procesa toda la secuencia bidireccionalmente.
    • Ideal para: ClasificaciÃ³n, NER, Question Answering extractivo.
    • Ejemplos: BERT, RoBERTa, ALBERT, DistilBERT.

    Decoder-only (GPT)

    • Genera texto de izquierda a derecha (autoregresivo).
    • Ideal para: GeneraciÃ³n de texto, completado.
    • Ejemplos: GPT-2, GPT-3, GPT-4, LLaMA, Mistral.

    Encoder-Decoder (T5)

    • Arquitectura completa original.
    • Ideal para: TraducciÃ³n, resumen, seq2seq.
    • Ejemplos: T5, BART, mBART.

    7.4. BERT (Bidirectional Encoder Representations from Transformers)

    BERT fue un hito en NLP al introducir el preentrenamiento bidireccional.

    Preentrenamiento

    BERT se preentrena con dos tareas:

    1. Masked Language Model (MLM): Predecir tokens enmascarados.
