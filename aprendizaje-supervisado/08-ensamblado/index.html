    Unidad 8. Algoritmos de Ensamblado (Ensemble Learning)

    Los Algoritmos de Ensamblado (Ensemble Methods) son una t茅cnica de Machine Learning que combina las predicciones de m煤ltiples modelos base (conocidos como weak learners o aprendices d茅biles) para construir un modelo final m谩s robusto y preciso (strong learner).

    8.1. Conceptos Clave y Categor铆as

    El objetivo principal es reducir el sesgo (bias) o la varianza (variance), o ambos. Los m茅todos de ensamblado se dividen principalmente en tres categor铆as seg煤n c贸mo combinan los modelos:

    3. Boosting: Se entrena el mismo algoritmo de forma secuencial. Cada nuevo modelo intenta corregir los errores cometidos por el modelo anterior. Su objetivo es reducir el sesgo (evitar underfitting). Ejemplos: AdaBoost, XGBoost.

    8.2. Voting Classifiers (Votaci贸n)

    Es la forma m谩s simple de ensamblado. Consiste en agregar las predicciones de clasificadores totalmente diferentes.

    Tipos de Votaci贸n

    • Soft Voting (Votaci贸n Suave): Si los clasificadores pueden estimar probabilidades (tienen m茅todo predict_proba), se promedian las probabilidades de cada clase. La clase con el promedio de probabilidad m谩s alto gana. El Soft Voting suele funcionar mejor porque da m谩s peso a los votos con "alta confianza".

    8.3. Bagging y Random Forest

    Bagging (Bootstrap Aggregating) implica entrenar el mismo algoritmo en diferentes subconjuntos aleatorios del dataset de entrenamiento.

    • Bootstrap: El muestreo se hace con reemplazo (una misma muestra puede aparecer varias veces en el mismo subconjunto).
    • Pasting: El muestreo se hace sin reemplazo.

    Una vez entrenados, los modelos agregan sus predicciones (moda para clasificaci贸n, promedio para regresi贸n).

    Random Forest (Bosques Aleatorios)

    Es una implementaci贸n espec铆fica y optimizada de Bagging usando rboles de Decisi贸n.     Hiperpar谩metros Clave:

    • n_estimators: N煤mero de 谩rboles (m谩s es mejor, pero m谩s lento).
    • max_features: N煤mero m谩ximo de caracter铆sticas a considerar en cada divisi贸n.
    • bootstrap: Si usar muestreo con reemplazo (True por defecto).
    • n_jobs: N煤mero de n煤cleos de CPU a usar (-1 para todos).

    Ejemplo en Python (Random Forest)

    8.4. Boosting (Impulso)

    El Boosting entrena predictores secuencialmente, cada uno intentando corregir a su predecesor.

    8.4.1. AdaBoost (Adaptive Boosting)

    El algoritmo presta m谩s atenci贸n a las instancias de entrenamiento que el predecesor clasific贸 incorrectamente.

    1. Entrena un clasificador base.
    2. Aumenta el peso relativo de las instancias mal clasificadas.
    3. Entrena un segundo clasificador con los pesos actualizados.
    4. Repite el proceso.

    Hiperpar谩metros Clave:

    • learning_rate: Cu谩nto contribuye cada modelo. Un valor bajo requiere m谩s estimadores.

    Ejemplo Python (AdaBoost):

    8.4.2. Gradient Boosting Machine (GBM)

    En lugar de ajustar los pesos de las instancias, GBM intenta ajustar el nuevo predictor a los errores residuales (la diferencia entre el valor real y el predicho) del predictor anterior.

    Ejemplo Python (GradientBoosting de sklearn):
